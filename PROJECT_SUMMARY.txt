3. DATA CLEANING & TRANSFORMATION
================================================================================

3.1 Z-SCORE COLUMNS (HW5, HW70, HW8)
  ────────────────────────────────────
  Raw Format: Integer × 100 (stored as small integers)
  
  Transformation:
    1. Convert to numeric, replace missing codes (9998, 9980, 9996, 9997, 9999) → NaN
    2. Divide by 100 to get actual Z-scores
    3. Remove biological outliers (|Z| > 6)
  
  Example:
    - Raw: -146 → -146/100 = -1.46 (valid HAZ)
    - Raw: 500 → 500/100 = 5.0 (valid WHZ)
  
  Final Ranges:
    - WHZ (HW5): -6.00 to +6.00 (expected: -3 to +3 typically)
    - HAZ (HW70): -6.00 to +6.00 (expected: -2 to +2 typically)

3.2 WEIGHT COLUMN (HW3)
  ──────────────────────
  Raw Median: 863 → interpreted as stored as ×100
  
  Transformation:
    1. Divide by 100 → 8.63 kg (plausible for ~2-year-old child)
    2. Remove outliers (< 1.5 kg or > 30 kg as invalid)
  
  Final Range: 1.5 to 30 kg

3.3 HEMOGLOBIN COLUMN (HW56)
  ───────────────────────────
  Raw Median: 103 → stored as ×10 (g/dL × 10)
  
  Transformation:
    1. Divide by 10 → 10.3 g/dL (normal for children)
    2. Remove outliers (< 3 or > 20 g/dL)
    3. Replace missing codes (999, 9998, 9999) → NaN
  
  Final Range: 3.0 to 20.0 g/dL

3.4 ANEMIA CLASSIFICATION (HW57)
  ───────────────────────────────
  Original Encoding: 1=not anaemic, 2=mild, 3=moderate, 4=severe
  Cleaned Encoding: 0=not anaemic, 1=mild, 2=moderate, 3=severe
  
  Remap: {1: 0, 2: 1, 3: 2, 4: 3}
  Missing Codes: 8, 9 → NaN

3.5 BINARY FLAGS (HW6, HW7, HW71, HW72)
  ──────────────────────────────────────
  Meaning:
    - HW6: Wasted (WHZ < -2)
    - HW7: SAM - Severe Acute Malnutrition (WHZ < -3)
    - HW71: Stunted (HAZ < -2)
    - HW72: Underweight (WAZ < -2)
  
  Encoding: 0=No, 1=Yes
  
  Strategy: If original flags were missing or invalid, re-derive from Z-scores
    Example: HW6 = (HW5 < -2).astype(int)

3.6 HEALTH HISTORY (H11, H22, H43)
  ─────────────────────────────────
  H11 (Fever):
    - Raw: 0=No, 2=Yes
    - Remap: {0: 0, 2: 1} → standard 0/1
    - Missing: 8, 9 → NaN
  
  H22 (Vitamin A):
    - Raw: 0=No, 1=Yes ✓ (already correct)
    - Missing: 8, 9 → NaN
  
  H43 (Deworming):
    - Raw: 0=No, 1=Yes ✓ (already correct)
    - Missing: 8, 9 → NaN
  
  Age-Aware Imputation:
    - For children < 6 months: Food recall not administered → fill with 0
    - For children < 12 months: Deworming not given → fill with 0

3.7 SEX COLUMN (B4)
  ──────────────
  Encoding: 1=Male, 2=Female
  Final: 0=Male, 1=Female (for compatibility)
  
  Struggle: Column location took effort to find (B4 from birth history series)

3.8 FOOD/DIETARY RECALL (V414A-V414S)
  ────────────────────────────────────
  13 food group indicators:
    - V414A: Grains
    - V414E: Eggs
    - V414F: Vitamin A-rich vegetables
    - V414G: Other vegetables
    - V414I: Other fruits
    - V414J: Organ meat
    - V414K: Processed foods
    - V414L: Sweet drinks
    - V414M: Nuts/seeds
    - V414N: Breastmilk
    - V414O: Formula milk
    - V414P: Thin porridge
    - V414S: Other foods
  
  Encoding: 0=Not consumed, 1=Consumed (in last 24h)
  Missing Codes: 8, 9 → NaN
  
  Age-Aware Imputation:
    - For children < 6 months: Often no recall → fill 0 (not yet eating)
    - For all: Missing → fill 0 (not consumed)

3.9 CATEGORICAL: WEALTH & EDUCATION
  ──────────────────────────────────
  V190 (Wealth Index): 1-5 scale (1=poorest, 5=richest)
  V106 (Mother Education): Ordinal scale (0=no education, ..., 4=higher)
  
  Strategy: Fill missing with mode (most frequent value)

5. FEATURE ENGINEERING & TARGET VARIABLES
================================================================================

5.1 MALNUTRITION TYPE (malnutrition_type)
  ───────────────────────────────────────
  Definition by Z-score thresholds:
    
    IF WHZ < -3 THEN 'SAM' (Severe Acute Malnutrition)
    ELSE IF WHZ >= -3 AND WHZ < -2 THEN 'MAM' (Moderate Acute Malnutrition)
    ELSE IF HAZ < -2 THEN 'Stunted' (Linear growth failure)
    ELSE 'Normal' (No acute malnutrition)
  
  Distribution:
    - Normal: ~70%
    - Stunted: ~20%
    - MAM: ~6%
    - SAM: ~4%

5.2 DEFICIENCY TYPE (deficiency_type) ← MAIN CLASSIFICATION TARGET
  ──────────────────────────────────────────────────────────────
  Three major deficiency pathways:
  
  1. Protein-Energy Malnutrition (PEM):
     Indicator: WHZ < -3 (SAM threshold)
     Rationale: Most severe acute malnutrition indicates inadequate energy/protein
  
  2. Iron Deficiency Anemia:
     Indicator: Anemia Level (HW57) >= 2 (moderate-severe) OR Hemoglobin < 10 g/dL
     Rationale: Low hemoglobin is direct measure of iron deficiency anemia
  
  3. Vitamin A Deficiency:
     Indicator: No Vitamin A supplement (H22=0) AND Stunted (HAZ < -2)
     Rationale: Stunting + no supplementation indicates high Vitamin A risk
  
  Priority Logic (Hierarchical):
    IF pem_risk == 1: 'Protein_Energy'
    ELSE IF iron_risk == 1: 'Iron_Anaemia'
    ELSE IF vita_risk == 1: 'Vitamin_A'
    ELSE: 'None'
  
  Note: Child can have multiple deficiencies, but we use priority classification

5.3 SEVERITY SCORE (y_score) ← REGRESSION TARGET
  ──────────────────────────────────────────────
  Composite score 0-100 combining three components:
  
  Component 1: WHZ Severity (0-40 points)
    - Formula: max(0, -WHZ) × 10, capped at 40
    - Example: WHZ = -2.5 → (2.5 × 10) = 25 points
  
  Component 2: HAZ Severity (0-30 points)
    - Formula: max(0, -HAZ) × 7, capped at 30
    - Example: HAZ = -2.0 → (2.0 × 7) = 14 points
  
  Component 3: Anemia Severity (0-30 points)
    - HW57=0 → 0 points (no anemia)
    - HW57=1 → 10 points (mild)
    - HW57=2 → 20 points (moderate)
    - HW57=3 → 30 points (severe)
  
  Total: Sum all components, capped at 100
  Example: WHZ=-2.5, HAZ=-2.0, HW57=1 → 25+14+10 = 49 points

5.4 FEATURE ENGINEERING
  ──────────────────────
  New Features Created:
  
  Demographic:
    - age_months: Child's age in months (cleaned from HW1)
    - sex: 0=male, 1=female
    - rural: 0=urban, 1=rural (from V025)
  
  Anthropometric:
    - weight_kg: Body weight in kg
    - whz: Weight-for-Height Z-score
    - haz: Height-for-Age Z-score
    - haemoglobin: Blood hemoglobin level (g/dL)
    - anaemia_level: Classification 0-3
  
  Socioeconomic:
    - wealth_index: 1-5 quintile
    - mother_edu: Education level (ordinal)
  
  Environmental:
    - clean_fuel: 1 if modern cooking fuel, 0 otherwise
    - improved_water: 1 if improved water source, 0 otherwise
    - improved_sanitation: 1 if improved toilet, 0 otherwise
  
  Health:
    - fever_2wks: Recent fever (0/1)
    - vit_a_suppl: Vitamin A supplement (0/1)
    - deworming: Deworming given (0/1)
  
  Dietary:
    - food_grain, food_legume, food_dairy, food_flesh, food_egg,
      food_vita_veg, food_other_veg, food_vita_fruit, food_other_fruit, etc.
    - dietary_diversity: Sum of 8 key food groups consumed (0-8 score)

Step 1: Filter usable data

8. KEY CHALLENGES FACED & SOLUTIONS
================================================================================

Challenge 1: Column Discovery & Naming Conventions
  ──────────────────────────────────────────────
  Problem:
    - NFHS/DHS data uses cryptic column names (HW5, V106, HC61, etc.)
    - Initial column list was incomplete (missing wealth, education, sex)
    - Had to manually search through 1,644 columns to find equivalent columns
  
  Solution:
    - Used metadata to print all column names
    - Cross-referenced DHS documentation
    - Final columns: V190 (wealth), V106 (education), B4 (sex)
    - Created mapping file to document all transformations

Challenge 2: Z-Score Encoding & Decimal Shifts
  ────────────────────────────────────────────
  Problem:
    - WHZ and HAZ stored as integer × 100 (e.g., -146 = -1.46)
    - Weight stored as kg × 100 (e.g., 863 = 8.63 kg)
    - Hemoglobin stored as g/dL × 10
    - HW8 was initially thought to be height in cm, but was actually another Z-score
  
  Solution:
    - Examined raw min/max/median to infer scaling
    - Compared medians to biological norms (e.g., 8.63 kg for ~2-year-old)
    - Applied appropriate division (÷100 for Z-scores, ÷10 for Hb)
    - Validated with domain knowledge of child anthropometry

Challenge 3: Missing Data Codes
  ──────────────────────────────
  Problem:
    - NFHS uses multiple missing codes: 8, 9, 999, 9998, 9999, 9980, 9996, 9997
    - Different columns use different codes
    - Easy to miss some and have invalid values in final dataset
  
  Solution:
    - Created mapping of which codes apply to which columns
    - Applied systematic replacement before scaling
    - Validated with value_counts() to ensure no invalid codes remain

Challenge 4: Age-Aware Imputation
  ────────────────────────────────
  Problem:
    - Not all children in survey get asked all questions
    - Children < 6 months don't get dietary recall
    - Children < 12 months don't get deworming
    - Simple median/mode imputation would create false values
  
  Solution:
    - First identified age thresholds from NFHS documentation
    - Applied age-specific rules BEFORE general imputation
    - Example: For children < 6m, set food items to 0 (developmentally logical)
    - General imputation (median/mode) only on remaining NaN values

Challenge 5: Column Type Conversions
  ──────────────────────────────────
  Problem:
    - pyreadstat loads columns as mixed types (float, int, object)
    - Some categorical columns had string values or were labeled
    - Direct comparisons failed when mixing types
  
  Solution:
    - Applied pd.to_numeric(col, errors='coerce') to all columns first
    - This converts everything to numeric and invalid values → NaN
    - Then applied domain-specific transformations
    - Final validation with value_counts()

Challenge 6: Deficiency Type Classification
  ──────────────────────────────────────────
  Problem:
    - Multiple possible deficiencies per child (wasting + anemia + Vit A risk)
    - Needed single target variable for classification
    - Different severity levels for each deficiency type
  
  Solution:
    - Created 3 separate risk indicators: pem_risk, iron_risk, vita_risk
    - Used hierarchical priority (PEM > Iron > Vitamin A)
    - Created severity_score as regression target capturing all dimensions
    - Both approaches allow flexibility for future modeling strategies

Challenge 7: Feature Selection from 1,600+ Columns
  ───────────────────────────────────────────────
  Problem:
    - Loading entire dataset not feasible (1,644 columns, ~8,000 rows)
    - Manually selecting relevant columns required domain knowledge
    - Easy to miss important features
  
  Solution:
    - Organized columns by domain:
      * Demographic: age, sex, location
      * Anthropometric: weight, height, z-scores
      * Health history: fever, supplements, deworming
      * Environmental: water, sanitation, fuel
      * Dietary: 13 food groups
    - Used logic: features predictive of child nutrition
    - Created feature_columns.pkl for reproducibility

Challenge 8: Imputation Order & Data Leakage
  ──────────────────────────────────────────
  Problem:
    - Simple approach: split then impute on each set separately
    - Correct approach: fit imputer on TRAIN only, apply to all sets
    - First attempt violated this principle → potential data leakage
  
  Solution:
    - Implemented proper order: Split FIRST → Fit imputers on TRAIN ONLY →
      Apply to all (val & test)
    - Used joblib to save imputers for production use
    - Validated that no information from val/test influenced TRAIN statistics

Challenge 9: Unbalanced Classes
  ────────────────────────────
  Problem:
    - 'None' (healthy): ~70% of children
    - 'Iron_Anaemia': ~15%
    - 'Vitamin_A': ~10%
    - 'Protein_Energy': ~5%
    - Severe class imbalance affects model training
  
  Solution:
    - Stratified splits maintain imbalance across train/val/test
    - Documented imbalance for future modeling (may use SMOTE, class weights)
    - Severity_score regression may be alternate approach (continuous target)
    - Classification metrics (F1, precision-recall) more important than accuracy

Challenge 10: Feature Dimensionality
  ─────────────────────────────────
  Problem:
    - After feature engineering: 40+ features
    - Some may be redundant or highly correlated
    - Risk of overfitting in small-medium dataset
  
  Solution:
    - Kept all features generated from domain logic
    - Documented correlations (Z-scores, flags are derived from same source)
    - Recommended feature selection (Boruta, permutation importance) in model.py
    - Separate dietary features may be combined via dietary_diversity score

12. GLOSSARY OF ABBREVIATIONS
================================================================================

DHS    - Demographic & Health Survey
NFHS   - National Family Health Survey (India)
Z-score - Standardized score (distance from mean in standard deviations)
WHZ    - Weight-for-Height Z-score (acute malnutrition indicator)
HAZ    - Height-for-Age Z-score (stunting/chronic malnutrition indicator)
WAZ    - Weight-for-Age Z-score (underweight indicator)
SAM    - Severe Acute Malnutrition (WHZ < -3)
MAM    - Moderate Acute Malnutrition (-3 ≤ WHZ < -2)
PEM    - Protein-Energy Malnutrition
CSV    - Comma-Separated Values
PKL    - Pickle (Python serialization format)
NaN    - Not a Number (missing value)
Mode   - Most frequent value in a distribution
Median - Middle value in sorted distribution
Stratified - Maintaining class proportions across splits
